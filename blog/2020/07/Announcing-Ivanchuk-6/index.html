Announcing Ivanchuk-6

Jul 27, 2020 by chessbyte
  • releases
  • announcements

ManageIQ Ivanchuk-6 is now available. This release contains bug fixes, UI tweaks, and stabilization. Here are just some of the things added since the Ivanchuk-5 release:

Fixed

  • Allow embedded_ansible role on multiple appliances (#19162)
  • Fixed cockpit process starting (#20077)
  • Addressed Possible Information Leak / Session Hijack Vulnerability by using the proper version of the rack gem (#20210)
  • [API] Add name to valid edit attributes for vms (ManageIQ/manageiq-api#832)
  • [AWS] Check instance exists before getting status during provisioning (ManageIQ/manageiq-providers-amazon#604)
  • [AWS] Update Floating IP target and event collection (ManageIQ/manageiq-providers-amazon#618)
  • [Ansible Tower] Catch exceptions when parsing job_templates (ManageIQ/manageiq-providers-ansible_tower#218)
  • [Automation Engine] Add cc and bcc into builtin method send_email’s inputs fields (ManageIQ/manageiq-automation_engine#404)
  • [Classic UI] Provision Dialog (prov_dialog) customize - do not fail when no :sysprep_enabled (ManageIQ/manageiq-ui-classic#6953)
  • [Classic UI] Remove Tags from Network Topology View and others so as to limit the ability for users to see other users’ tags (ManageIQ/manageiq-ui-classic#6351)
  • [Classic UI] Fix cancel, save and select_tree actions in breadcrumbs and title in PXE menu section (ManageIQ/manageiq-ui-classic#7045)
  • [Classic UI] Fix breadcrumbs for Dashboard Widgets (ManageIQ/manageiq-ui-classic#7063)
  • [Classic UI] Add reconfigureMode parameter to dialog-user (ManageIQ/manageiq-ui-classic#7074)

You can download the Ivanchuk-6 release here.
